3Logic Flow

Read sectors from scenarios.py (thermal/steel/cement)→ Choose strategy: author original (Pillow)→ Write generator to the sprite spec→ Run → 13 PNGs→ Verify opacity + contact sheet; update ATTRIBUTIONS

4Math / Algorithm Used

Isometric cuboid projection on a shared 64×32 diamond footprint (half-width 32, half-height 16) centred at sprite (32, 46). Each box draws three faces — top (×1.18 brightness), left (×0.96), right (×0.74) — for a consistent top-left light. All sprites share the footprint so they align on the isocity.js grid (TILE_W=64, TILE_H=32).

5Tools / Formulas / Methods

  • Pillow 12.1 via scripts/gen_building_sprites.py — polygon faces + outline, sector features (stacks/silos/panels).
  • Grep on engine/scenarios.py / engine.py — confirmed sectors and that sector_label = sector.replace("_"," ").title() ("Thermal Power" / "Steel" / "Cement").
  • Verification: opaque-pixel counts + a 3× contact sheet (visually inspected).

7Limitations / Second-Best Alternative

  • Deviation from the plan's default. The Sprint-2 default was "start from CC0/CC-BY packs." Binary art packs can't be fetched into the repo in this environment, so I authored original sprites via script instead — which also removes all IP risk and is reproducible. Hand-crafted/commissioned art can drop in later via the same manifest.
  • Procedural, not artisan. Sprites are clean geometric iso buildings, not hand-pixelled Kairosoft art; dirty/clean differences are accent-led (tip/panel colors) and lean on the smog overlay.
  • Second-best: SVG sprites — rejected to keep the existing 64×64 PNG raster pipeline unchanged.

8Errors / Warnings / Flags Encountered

  • Strategy deviation flagged (author-original vs CC0 packs) — see Limitations.
  • Pillow getdata deprecation warning in the verify snippet only — not in shipped code.
  • No errors generating or saving sprites.
