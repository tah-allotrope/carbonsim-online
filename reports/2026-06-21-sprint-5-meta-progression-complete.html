Sprint 5 — Meta: Capex Abatement, XP Unlock Tree & Jurisdiction Skins

The last sprint adds the persistent layer: abatement becomes a capex asset you can finance with loans, technology can fail, XP feeds a Kairosoft-style unlock tree saved across games, and the whole simulation can re-skin to EU ETS or California. All of it lands without disturbing the carefully-tuned Sprint 1–4 balance.

Section 2

Logic Flow

The guiding constraint shaped every decision: the new mechanics are additive ledgers. active_abatement_ids remains the single source of truth for emissions, so the projection and penalty math are byte-identical to Sprint 4 on a default measure. Capex, opex, loans, and tech-failure ride alongside and are serviced once per year.

Section 3

Math / Algorithm Used

Loan amortization (finance_abatement)

term  = min(asset_life_years, scenario_loan_term)
# standard annuity payment
annual_payment = capex * r / (1 - (1 + r)^(-term))      # r = scenario_interest_rate
# r == 0 fallback: capex / term

Tech failure (seeded, reproducible)

rng = Random(f"{game_seed}-techfail-{company_id}-{year}")
for asset in active_abatement_assets:
    if asset.tech_risk_pct > 0 and rng.random() < asset.tech_risk_pct:
        company.tech_impaired_ids.append(asset.measure_id)   # 1 year only
# in _projected_emissions: impaired measures deliver 50% of abatement_amount

XP & levels

XP_EVENT_POINTS = {penalty_free_year:50, first_compliance:100,
                   otc_trade_closed:20, vcm_project_matured:40, clean_game:250}
XP_LEVEL_THRESHOLDS = [0, 200, 500, 1000, 2000, 4000]
xp_level = count(t in thresholds where xp >= t)

Jurisdiction balance tuning

Each jurisdiction has different offset economics (EU: cap 5%, $75/t; California: cap 8%, $38/t), which moves the penalty at which strategies balance. The dominance sweep was re-run per jurisdiction and penalty_rate tuned until no strategy exceeded 60% win rate — the same method established in Sprint 3.

Section 4

Tools / Formulas / Methods

  • Build-time enrichment — _enrich_measure adds capex fields with balance-neutral defaults at company construction, so no scenario content had to be hand-edited.
  • Deterministic RNG — tech-failure rolls derive from the game seed + a salt, keeping the determinism guarantee intact.
  • SQLite — a new player_profiles table on the existing mayor.db; XP stored monotonically so a lower in-session value can't erase lifetime progress.
  • FastAPI — new player router (GET/POST /api/players/{name}/xp) registered in main.py; finance_abatement added to the decision allow-list.
  • JSON overlays — jurisdiction skins as standalone data files merged over the base pack via load_jurisdiction.
  • Monte-Carlo sweep — run_strategy_sweep(jurisdiction=...) for the final cross-jurisdiction balance gate.
  • Browser preview — Unlock Tree modal, EU ETS game, and the loan path all verified live; zero console errors.

Section 6

Limitations / Second-Best Alternative

  • Capex defaults are balance-neutral by design. Existing measures keep capex == cost, zero opex, zero tech-risk — so the feature is fully wired but only bites where a measure or jurisdiction opts into richer values. The rejected second-best was retro-fitting opex/risk onto every existing measure, which would have silently re-balanced four sprints of tuning.
  • Unlock tree displays, but does not yet hard-gate every feature. The tree, levels, and persistence are real; wiring each node to actually toggle its mechanic (limit orders, 2-year forwards) is follow-on content work, not new plumbing.
  • Two XP systems coexist. The shipped client-side Progression drives the HUD bar; the new canonical server XP drives the unlock tree and persistence. They are intentionally separate; reconciling them into one is a tidy-up for later.
  • Bots ignore financing. AI agents still pay capex in cash via activate_abatement; they never take loans or react to tech-failure. The sweep is therefore conservative about the loan mechanic.
  • Jurisdictions reuse the base abatement catalog. EU/California companies map onto the existing three sectors; bespoke sector mixes would be a content addition.
Section 7

Errors / Warnings / Flags Encountered

Flag — jurisdictions broke the dominance gate at first
The first EU ETS / California sweeps failed TASK-05-11: EU favored aggressive (0.70), California favored conservative (0.75). Root cause was expected — each jurisdiction's distinct offset cost shifts the penalty at which strategies balance. Resolved exactly as in Sprint 3 by re-tuning each jurisdiction's penalty_rate (EU 1100→950, CA 850→1100); both now sit under 60%.
Bug (test) — tech-failure restore timing
The tech-failure test initially asserted restoration one phase-cycle too early; impairment clears at the next _start_year, not the next force_advance. Corrected the test's phase advancing — the engine behaviour was right.
Observation — shared modal centering quirk
The new Unlock Tree modal measured off-screen in the headless preview — but the shipped Save modal behaves identically, so it is a pre-existing shared .modal-overlay quirk, not a Sprint 5 regression. Content renders correctly (verified by scrolling the modal into view).
Section 8

Open Questions / Next Phase Seeds

  • Hard-gate unlock nodes. Wire each unlocked node to actually enable its mechanic (auction limit orders, 2-year forwards, speculator AI) rather than just display unlocked status.
  • Teach bots to finance. Let CompanyAgent choose loans when cash-constrained and react to tech-failure — would make the loan/opex economics matter in the sweep.
  • Unify XP. Collapse the client Progression heuristic and the server XP engine into one canonical source feeding both the HUD and the tree.
  • Jurisdiction selector UI. The backend accepts jurisdiction on game create and the skins are balanced; a new-game selector on index.html would surface it to players.
  • Capex/loans/assets panel. Asset life, opex, and outstanding loans are in the snapshot but not yet shown; a finance panel would make the capex decision legible.
  • Star rating & binary win gate (DEC-002). The meta layer is in place; the end-game star rating that ties XP, penalties, and survival together is the natural capstone.
