Section 2

Logic Flow

Each bot is wrapped in a CompanyAgent. plan_year is near-pure — it reads state and returns an ordered action list without mutating anything; run_bot_turns is the dispatcher that applies each action through the normal reducer so every bot move is audited exactly like a human move. A second pass lets bot buyers respond to OTC proposals raised this turn.

Horizon awareness comes from _estimate_future_gap: it projects next year's emissions (growth, keeping current abatement) against next year's free allocation (scaled by the public allocation-factor decline). Agents with horizon_years > 1 pre-buy forwards/VCM to cover part of that anticipated shortfall; a 1-year planner returns 0 and skips hedging.

Section 3

Math / Algorithm Used

Abatement ranking & horizon gate

Candidate measures are ranked by cost-effectiveness cost / max(abatement_amount, 0.01) (cheapest tonne first, ≤10 evaluated per RISK-03-01). A measure is skipped when its per-tonne cost exceeds the penalty avoided over the horizon — i.e. when cost/tonne > penalty_rate × horizon_years — unless the agent is aggressive (which abates regardless). Conservative agents stop once the running gap falls to projected × (1 − abatement_threshold_fraction).

Future-gap estimate (horizon lookahead)

next_emissions = max(baseline × (1+growth)^(next_year-1) − active_cut, 0)
next_alloc     = current_alloc × (next_factor / this_factor)
future_gap     = max(next_emissions − next_alloc, 0)        # 0 if horizon ≤ 1

Spot price seen by agents (Sprint 2 model, reused)

spot = base_price × (1 + ε × min(demand / supply_cap, 1))      # ε=0.4, S=50t
forward_locked = round(spot × 1.05, 2)                          # 5% term premium

OTC sizing & pricing

surplus  = allowances − effective_projected_emissions
quantity = min(surplus, peer_compliance_gap) × otc_appetite
price    = base_price × (1 + 0.3 × otc_appetite)               # speculator charges most

Sweep win metric — liquidation net worth

net_worth = cash                                  # already net of penalties + spend
          + banked_allowances × auction_floor
          + offset_holdings   × offset_price
win_rate(strategy) = wins / seeds  (winner = highest net_worth per game)

Net worth was chosen deliberately: ranking on penalties only rewards spending unlimited cash to zero them out (aggressive wins 90%); ranking on cash only rewards hoarding while eating penalties (conservative wins 100%). Net worth values converting spend into compliance assets and exposes the genuine balance question.

Section 4

Tools / Formulas / Methods

  • Python dataclass — CompanyAgent built from a company dict via from_company, reading its strategy profile from BOT_STRATEGIES.
  • Lazy imports — agents.py imports engine helpers inside methods, and engine.run_bot_turns imports CompanyAgent lazily, to break the circular dependency.
  • Greedy bounded planning — at most ~10 abatement candidates, one forward, one VCM, one OTC proposal per agent per year (RISK-03-01 mitigation).
  • Monte-Carlo sweep — head-to-head games assigning each of 5 strategies to one company; statistics.mean aggregates cash/penalties; deterministic per seed.
  • FastAPI — read-only GET /api/games/{id}/ai-signals returns each bot's strategy, posture, preferred instruments, and open-trade count.
  • pytest / unittest — 67 tests including a 20-seed in-suite dominance assertion.

Section 6

Limitations / Second-Best Alternative

  • Speculator never wins the sweep (0%). The bank-and-sell profile carries the most penalties ($688k mean) and tops net worth in zero of 20 seeds. It is balanced (well under 60%) but is currently the weakest profile — a candidate for buff in a later pass.
  • Penalty hike is a real difficulty change. Raising solo_standard from 325 to 1000 $/t makes the human game meaningfully harder. This is intentional (at 325, ignoring compliance was optimal, defeating the carbon theme) but only solo_standard was re-tuned; solo_easy and solo_hard were left as-is and may carry the same under-compliance incentive.
  • Win metric is a modelling choice. Liquidation net worth is one defensible objective; the live game leaderboard still ranks by cumulative penalties (DEC-002). The two can disagree. A future Sprint 5 star-rating could reconcile them.
  • Second-best AI design: a full BazaarBot multi-agent clearing market (ALT-001) was rejected — disproportionate complexity at 8 companies. The greedy bounded planner achieves varied, balanced behaviour without a global solver.
Section 7

Errors / Warnings / Flags Encountered

Flag — num_years=None override
First sweep run crashed: passing num_years=None into create_initial_state overrode its default of 3 with None. Fixed by resolving years from the scenario pack inside run_strategy_sweep when not supplied.
Flag — Windows console Unicode
print_strategy_sweep crashed on cp1252 when printing ⚠/✓/em-dash. Replaced with ASCII-safe markers ([!] / [ok]).
Bug fixed — shared mutable scenario state (root cause)
Sweep results were unstable across repeated calls in one process: the same penalty=900 gave 0.45 then 0.70. Root cause: state["allocation_factors"] aliased the shared SCENARIO_PACKS[...]["allocation_factors"] dict, and apply_shock("election_pressure") rewrites it in place — so one game's policy shock permanently corrupted the cap schedule for every later game in the process (and silently changed difficulty). Fixed by deepcopy-ing allocation_factors in create_initial_state. The sweep is now stable across repeated calls, which also meant re-tuning the penalty on the correct baseline (900 still dominant → 1000).
Resolved — balance regime, not a code error
The sweep surfaced that at the inherited penalty_rate of 325, a pure under-complier dominated every metric. Treated as a design finding and corrected via the documented penalty re-tune (TASK-03-07 explicitly authorizes this).
Section 8

Open Questions / Next Phase Seeds

  • Re-tune easy/hard? Should solo_easy (penalty 201) and solo_hard (520) get the same compliance-rationality check, or is under-compliance acceptable on easy by design?
  • Buff the speculator? At 0% sweep wins it is the weakest profile. Worth raising its abatement floor or lowering its OTC premium so it is competitive without becoming dominant.
  • Surface AI signals in the UI. The /ai-signals endpoint exists but is not yet rendered. A competitor-posture panel in web/game.html would add strategic texture (Sprint 4 candidate).
  • Sprint 4 (Story) dependency. Cascading cards + policy_stability will feed agents new shocks; plan_year already tolerates dynamic caps, but agent reactions to a regulatory_crackdown should be validated once cards exist.
  • Auctions in live solo play. The sweep opens/closes auctions each year; the solo REST flow still leaves them dormant. Wiring auction open/close into advance-year would let the player and agents actually compete for auctioned allowances.
