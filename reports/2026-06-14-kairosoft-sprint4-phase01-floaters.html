1Phase Header

First phase of Kairosoft Sprint 4 — Floating Feedback Juice (plans/2026-06-14-kairosoft-feedback-juice-plan.md). Goal: add a rising/fading on-canvas feedback layer (coins / numbers / reaction emotes) anchored to the player's plot, exposed as a public API for event wiring in Phase 2.

2Input → Output Summary

InputOutput
Only particle bursts + DOM toasts; no spatial pop-ups Isocity.float(text,color) (outlined rising number/coin) and Isocity.emote(symbol) (rising emoji) on the city canvas, anchored above the player plot; updated in the loop, drawn above buildings/citizens

3Logic Flow

float/emote push to floaters[]→ updateFloaters (rise + fade)→ drawFloaters (layer 5, above markers)→ expose float/emote in API

4Math / Algorithm Used

Each floater rises (vy ≈ −0.55 px/frame; emotes slower) and fades (life −= 0.012/frame, ~2.8s) then is culled. Capped at 24. Numbers get a white stroke + colored fill for legibility on any background; emotes render as emoji glyphs.
