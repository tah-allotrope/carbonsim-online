Bright Kairosoft Palette Tokens

CarbonSim Online · repo carbonsim-online · 2026-06-14

1Phase Header

First phase of the Kairosoft Visual Upgrade — Sprint 1 (Bright Palette & Rounded UI/Font) (plans/2026-06-14-kairosoft-palette-ui-plan.md). Goal: shift the whole app's mood from the muted retro-earth scheme to a bright, cheerful Kairosoft palette via a pure CSS token swap — the cheapest, highest-visibility lever, and the palette every later art sprint must match. Scope was the color/token layer only; bevel→rounded chrome and the display font are deliberately deferred to Phase 02 to keep the phases cleanly separable.

3Logic Flow

Read full style.css→ Retune core color tokens (keep semantic names)→ Add --accent-bright, gradient & --radius-lg tokens→ Reload preview (cache-bust)→ Verify computed tokens + menu screenshot

Token names were held stable so the swap propagates to every screen with zero markup edits.

4Math / Algorithm Used

No algorithm. The only quantitative check was WCAG contrast sanity: dark --text #2a2018 on --bg #fdf6e9 is high-contrast; saturated hues were reserved for accents/buttons/badges where text color (white or near-black) keeps legibility.

5Tools / Formulas / Methods

  • Edit on web/css/style.css — single :root block.
  • Claude Preview — preview_start, cache-busted reload, preview_eval (computed-style readback), preview_screenshot.
  • Design principle: tokenized CSS → one source of truth; swap values, not markup.

7Limitations / Second-Best Alternative

  • Intentional mid-state: chrome is still beveled (hard 3-D edges) and the font is still blocky Press Start 2P — both addressed in Phase 02. Mid-phase the bright palette + old bevel borders look slightly transitional.
  • Accent vs. brightness tradeoff: a very bright azure would fail white-text contrast on buttons; #1f93c7 was chosen as bright-but-legible.
  • Second-best: a separate kairosoft.css overlay — rejected to avoid dual-skin maintenance; editing the single tokenized source is cleaner.

8Errors / Warnings / Flags Encountered

  • Preview server had to be restarted (it is tied to the session and drops between turns) — resolved with preview_start.
  • Browser served cached CSS until a cache-busting reload; the /css mount already sends Cache-Control: no-cache, so a normal hard-refresh suffices for users.
  • No CSS/build errors. pytest unaffected (frontend-only change).

9Open Questions / Next Phase Seeds

  • Open decision (Phase 02): keep Press Start 2P or add a rounded pixel display font. Default chosen for this sprint: add a rounded display font, keep mono for stats.
  • Phase 02 seeds: round panels/buttons/badges using --radius-lg + the new gradient tokens; replace hard bevel shadows with soft rounded shadows; add a reusable "notification bubble" style for Sprint 4.
  • Phase 03 seed: define the building sprite color spec (for Sprint 2) in docs/design-language.md against this palette.
