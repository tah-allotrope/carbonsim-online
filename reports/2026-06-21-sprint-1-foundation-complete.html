Math / Algorithm Used
Deck Seed Formula

Each game's card draw is now deterministic per game × year:

rng_seed = random.randint(0, 2³² − 1)
draw_rng = Random(rng_seed ^ (year × 2654435761 & 0xFFFFFFFF))

The Knuth multiplicative constant (2654435761) distributes year values uniformly across the seed space, preventing year=1 and year=2 from producing nearly identical sequences.

project_outcome: buy_offsets
offset_cap = projected × offset_cap_pct
current_usable = min(holdings, cap)
new_usable = min(holdings + qty, cap)
gap_delta = −(new_usable − current_usable)
cash_delta = −qty × offset_price

Correctly handles the cap ceiling: buying above the offset cap produces non-zero cash_delta but zero additional gap reduction — surfaced as a warning note.

Limitations / Second-Best Alternative
Limitation — Determinism Scope

check_determinism compares card_frequency as a proxy for full audit_log identity. It won't catch subtler divergence (e.g., different draw order producing the same frequency histogram). A deeper check would serialize and compare the full audit_log — deferred to avoid requiring audit_log surfacing through run_playtest.

Limitation — end_year Phase Guard

The end_year route was modified to call force_advance_phase once before capturing the year summary, then loops to drain compliance. If the game is already in compliance phase when end_year is called (edge case), the summary will be generated correctly; if it's in year_start already, the summary will be empty. Not a regression but a known edge.

Alternative — API-backed consequence preview

The plan specified GET /project + tooltip on hover. The implemented approach does the preview math client-side in JS using already-loaded game state — zero latency, no round-trip. The server endpoint is also wired for Sprint 3 AI agent use. The client-side path is strictly better for UX; the server path exists for programmatic access.

Alternative — LLM year-end narrative

Template-driven narrative was chosen over LLM generation. Trade-off: templates are deterministic and offline-capable but produce less varied text across many runs. LLM generation could be layered on as a Sprint 5 enhancement without changing the API contract (same year_summary field).
