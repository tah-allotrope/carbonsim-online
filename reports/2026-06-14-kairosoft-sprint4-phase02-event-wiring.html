1Phase Header

Wire the Phase-1 floating-feedback API to the events players care about so meaningful actions produce a little payoff.

3Logic Flow

doDecision → emote + coins (lookup measure cost)→ render compliance-change → 😊/😡→ syncProgressionState → level-up flourish→ verify on real click

4Math / Algorithm Used

None. Event hooks call the Phase-1 float/emote API; level-up detected by comparing the new level to the previously stored one.

5Tools / Formulas / Methods

  • Edit web/game.html — doDecision (abatement/offset feedback, measure lookup for cost/reduction), compliance-change block (😊/😡), syncProgressionState (level-up).
  • Kept the existing toast for textual confirmation; emotes/coins are the spatial layer (no duplicate signaling for the same fact).
  • Claude Preview — clicked the real Activate button; full-canvas pixel diff.

7Limitations / Second-Best Alternative

  • Feedback anchors to the player plot only (snapshot limitation — bots' actions aren't surfaced).
  • Offset cost coin omitted (client doesn't reliably know the scenario offset price); shows quantity instead.
  • Second-best: emote on every click — avoided; reserved for meaningful state changes to prevent noise.

  • None. Real-click path fires feedback; no console errors.
