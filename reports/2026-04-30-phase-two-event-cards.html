03 / Math / Algorithm Used

Math / Algorithm Used

Weighted Random Draw

Cards are drawn from eligible pool using a weighted random selection without replacement:

total = sum(weight for eligible cards not yet drawn)
r = rng.uniform(0, total)
for each card in eligible:
    running += card.weight
    if r <= running: pick this card
loop for count=3 cards

Prerequisite Filtering

Before drawing, the deck filters cards against current game state:

  • min_year / max_year — card only appears within a year range
  • min_cash — requires player cash above threshold
  • max_penalties — requires cumulative penalties below threshold

06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

Limitations

  • Card balance not tuned: weights and magnitudes are initial estimates. Phase 5 (playtest) will rebalance based on actual playthrough data.
  • No choice branch depth: choices are binary (accept/reject) with deterministic outcomes. Richer decision trees deferred to Phase 5.
  • fdi_proposal simplified: Instead of adding a new company/facility, it provides a cash infusion. Full portfolio expansion deferred to Phase 3+.
  • No card images/visuals: cards are text-only. Visual theming deferred to Phase 4 frontend.
  • No "card of the year" bottleneck: In Climate Mayor Phase 4, only the drawn cards matter; player doesn't choose from a hand.

Second-Best Alternative

An alternative approach would have been a pure JSON-driven effect system without new shock types, where card effects directly mutated the state dict. This was rejected because: (a) it duplicates the existing apply_shock dispatch, (b) it forces card resolution and facilitator-triggered shocks to use different code paths, and (c) the new shock types (tech_unlock, election_pressure, etc.) are genuinely useful for the facilitator workshop mode too.
