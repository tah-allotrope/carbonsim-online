1Phase Header

First phase of Kairosoft Sprint 3 — Animated Chibi Citizens (plans/2026-06-14-kairosoft-citizens-plan.md). Goal: give isocity.js the sprite-sheet frame-animation capability it lacked (it only blitted single frames) and author the chibi citizen sheet that Phase 2 will animate and move.

2Input → Output Summary

InputOutput
Renderer blits single static images; no character art drawSprite() (9-arg sub-rect blit, optional flip) + walkFrame() clock; citizens.png sheet (64×48: 4-frame walk × 3 shirt variants) registered with frame metadata

3Logic Flow

Author citizen sheet (Pillow)→ Register in manifest w/ frame meta→ Add drawSprite + walkFrame to isocity→ Load sheet + meta→ Verify slice + meta in browser

4Math / Algorithm Used

Frame index = floor(performance.now()/150) mod frames — wall-clock based so the walk cycle stays stable under the 30fps throttle and idles (frame 0) under reduced motion. Sub-rect blit reads cell (col·frameW, row·frameH) from the sheet.

5Tools / Formulas / Methods

  • Pillow — extended scripts/gen_building_sprites.py with make_citizens() (16×16 chibi: head/hair/eyes/shirt + alternating legs).
  • Edit web/assets/manifest.json (citizens entry + frame meta) and web/js/isocity.js (load sheet/meta, drawSprite, walkFrame).
  • node --check + Claude Preview (AssetLoader load, frame slice).
