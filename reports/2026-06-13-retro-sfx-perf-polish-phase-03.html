Sprint 5 Phase 03 Report

Accessibility Parity & Sign-off

Verified the retro tycoon look does not regress accessibility: reduced-motion static city, visible keyboard focus, and contrast-corrected palette across all four screens.

Date: 2026-06-13 Commit: c46fecd Tests: 92 passed

What Changed

Reduced-Motion Isometric City (web/js/isocity.js)

  • Added a runtime listener for prefers-reduced-motion changes.
  • When reduced motion is enabled, the RAF loop stops and a single static frame is drawn.
  • update() now re-renders the static city when reduced motion is active so reconnects/updates remain visible.
  • DOM particles are hidden via CSS when reduced motion is requested.

Global Focus States (web/css/style.css)

  • Added a global :focus-visible rule (2px solid --accent, 2px offset) covering links, buttons, inputs, selects, and tabbable elements.
  • Preserved component-specific focus styles for buttons and inputs.

Contrast Fixes

  • .year-wipe-title: white-on-parchment changed to var(--text).
  • .event-card .cat-market & .cat-opportunity: colored small text changed to var(--text) with semantic left border indicator.
  • .toast-success .toast-title & .toast-warning .toast-title: colored small text changed to var(--text); progress bars keep semantic colors.
  • .badge-green & .streak-badge: white-on-green small text changed to dark ink.
  • .ach-pop-xp: green text on panel changed to dark ink.
  • .achievement-popup-overlay: removed neon purple chrome; retokenized to retro panel/bevel colors.
  • .progress-bar, .xp-bar-outer, .timeline-segment, .compliance-progress-gauge .bg-circle: replaced leftover rgba(255,255,255,0.06–0.08) tracks with var(--line)/var(--bg-soft) for perceivability on parchment.

Reduced-Motion CSS (web/css/animations.css)

  • Replaced the narrow reduced-motion media query with a comprehensive rule that sets all animation/transition durations to 0.01ms and hides .visual-particle.

Documentation (docs/design-language.md)

  • Updated the Accessibility section with verified contrast ratios and the reduced-motion/focus behavior.
